# TEMU 看板服务器部署说明

## 最终架构

- 云服务器：Docker 跑 MySQL + Web 看板。
- 本机 Windows：领星爬虫和仓库库存刷新，每 4 小时写入云服务器 MySQL。
- 运营人员：访问 `http://服务器IP:3106`。

服务器不运行领星爬虫，不依赖浏览器登录态。

## 1. 服务器准备

建议服务器：

- Ubuntu 22.04 或 24.04。
- 2 核 4G 起步。
- 安全组开放 `3106` 给运营访问。
- `3306` 先不要开放公网；等本机爬虫写库时，只允许你的办公公网 IP 访问。

登录服务器：

```bash
ssh root@服务器IP
```

安装 Docker：

```bash
curl -fsSL https://get.docker.com | bash
systemctl enable docker
systemctl start docker
docker --version
docker compose version
```

## 2. 上传或拉取项目

推荐用 Git：

```bash
git clone 你的仓库地址
cd 项目目录/前后端价格显示及预警功能/modules/temu-price-dashboard
```

如果暂时不用 Git，也可以把整个项目文件夹上传到服务器，再进入同一个目录。

## 3. 创建服务器 .env

复制示例：

```bash
cp .env.server.example .env
```

编辑：

```bash
nano .env
```

示例：

```env
WEB_PORT=3106
MYSQL_BIND_HOST=127.0.0.1
MYSQL_PUBLIC_PORT=3306
DB_NAME=temu_monitor
DB_USER=temu_app
DB_PASSWORD=换成强密码
MYSQL_ROOT_PASSWORD=换成另一个强密码
```

说明：

- `WEB_PORT`：网页端口，默认 `3106`。
- `MYSQL_BIND_HOST=127.0.0.1`：MySQL 只给服务器本机访问，最安全。
- 如果本机爬虫要直接写服务器 MySQL，再改成 `MYSQL_BIND_HOST=0.0.0.0`，并在云服务器安全组里把 `3306` 限制为只允许你的办公公网 IP。
- `DB_PASSWORD` 和 `MYSQL_ROOT_PASSWORD` 不要一样。
- `.env` 不要提交到 Git。

## 4. 启动 MySQL + Web

```bash
docker compose -f docker-compose.server.yml --env-file .env up -d --build
```

第一次启动会做这些事：

1. 拉取 MySQL 镜像。
2. 构建 Web 镜像。
3. 创建 MySQL 数据卷 `temu_mysql_data`。
4. 初始化看板表结构。
5. 启动 Web 看板。

检查：

```bash
docker ps
docker logs --tail 100 temu-monitor-mysql
docker logs --tail 100 temu-price-dashboard
curl http://127.0.0.1:3106/api/health
```

访问：

```text
http://服务器IP:3106
```

刚部署完如果还没导入数据，页面可能提示暂无快照。等本机爬虫写入服务器 MySQL 后恢复正常。

## 5. 本机爬虫改写服务器数据库

本机文件：

```text
前后端价格显示及预警功能\modules\temu-price-dashboard\.env
```

把数据库地址改成服务器：

```env
DASHBOARD_HOST=0.0.0.0
DASHBOARD_PORT=3106
DATA_SOURCE=db
ENABLE_LOCAL_REFRESH=false
DB_HOST=服务器IP
DB_PORT=3306
DB_USER=temu_app
DB_PASSWORD=服务器.env里的DB_PASSWORD
DB_NAME=temu_monitor
```

同时云服务器安全组：

- `3106`：允许运营访问。
- `3306`：只允许你的办公公网 IP。

然后本机手动跑一次：

```powershell
cd "C:\Users\Administrator\Desktop\project\6\前后端价格显示及预警功能\modules\lingxing-temu-crawler"
powershell -ExecutionPolicy Bypass -NoProfile -File .\scripts\run_lingxing_fetch.ps1
```

跑完后在服务器检查：

```bash
curl http://127.0.0.1:3106/api/health
```

## 6. 日常更新代码

服务器：

```bash
cd 项目目录
git pull
cd 前后端价格显示及预警功能/modules/temu-price-dashboard
docker compose -f docker-compose.server.yml --env-file .env up -d --build
```

## 7. 数据库备份

备份：

```bash
docker exec temu-monitor-mysql mysqldump -uroot -p temu_monitor > temu_monitor.sql
```

恢复：

```bash
docker exec -i temu-monitor-mysql mysql -uroot -p temu_monitor < temu_monitor.sql
```

换服务器时：

1. 新服务器安装 Docker。
2. 拉项目。
3. 创建 `.env`。
4. 启动 MySQL + Web。
5. 恢复 `temu_monitor.sql`。
6. 本机爬虫 `.env` 改成新服务器 IP。

## 常用命令

```bash
docker ps
docker logs --tail 100 temu-price-dashboard
docker logs --tail 100 temu-monitor-mysql
docker compose -f docker-compose.server.yml --env-file .env restart
docker compose -f docker-compose.server.yml --env-file .env down
```
